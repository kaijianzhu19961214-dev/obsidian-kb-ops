# Weekly Synthesis Workflow

Goal: turn the week's notes, clips, reports, and experiments into a short review.

Scope: create one concise weekly knowledge note that links to source notes and extracts decisions, durable ideas, open loops, and next actions.

Default destination:

```text
/Users/zhukaijian/Documents/Obsidian Vault/60_Daily/YYYY-Www-weekly-synthesis.md
```

## Inputs

- daily notes in `60_Daily/`
- new or updated notes in `00_Inbox/`
- new source/report/data notes in `30_Resources/`
- active project notes in `10_Projects/`
- recent experiment summaries or failure notes
- external data watch report for the same week

## Sections

```text
本周主线：
本周新增知识：
重要结论：
量化研究启发：
数据源更新：
实验进展：
失败记录：
可晋升 Permanent Note：
下周行动：
需要清理的笔记：
待确认：
```

## Rules

- Prefer links to source notes over duplicated content.
- Mark uncertain claims as `待确认`.
- Separate facts from hypotheses.
- Keep the weekly note concise enough to review.
- Do not turn every source into a permanent note; promote only reusable ideas.
- Keep action items concrete and reviewable next week.

## Steps

1. Collect notes modified during the week.
2. Group them into:
   - captures
   - reports
   - external data
   - project work
   - experiments
   - durable ideas
3. Identify repeated themes and contradictions.
4. List project decisions and unresolved questions.
5. Create or update the weekly synthesis note.
6. Add backlinks from relevant MOCs only for high-value notes.
7. Carry forward unfinished actions with an explicit reason.

```text
Permanent 候选：

```
